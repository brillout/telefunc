This page covers the Cloudflare-specific setup for <Link text={<code>telefuncWebSocket()</code>} href="/telefuncWebSocket" />.


## Minimal setup

This works out of the box:

```ts
import { telefuncWebSocket } from 'telefunc/websocket/cloudflare'

const ws = telefuncWebSocket()

export const TelefuncDurableObject = ws.createDurableObjectClass()

export default {
  async fetch(request: Request, env: Env, ctx: ExecutionContext) {
    const resp = ws.handleTelefunc(request, env, ctx)
    if (resp) return resp
    // ... rest of your fetch handler
  },
}
```

Call `ws.handleTelefunc()` before the rest of your application logic so Telefunc can intercept its own RPC and WebSocket requests.

## Context

To pass per-request data to your telefunctions via <Link text={<code>getContext()</code>} href="/getContext" />, provide a `context` function:

```ts
const ws = telefuncWebSocket({
  context: async (request, env: Env) => ({
    user: await getUserFromCookie(request, env),
  }),
})
```

The function receives the incoming `Request` and your `env` bindings, and its return value is available inside telefunctions via `getContext()`.


## Scale

By default all channels share a single Durable Object instance per location bucket. If you need more capacity, increase the number of instances:

```ts
const ws = telefuncWebSocket({ scale: 4 })
```

Each browser session is pinned to one shard via an opaque token stored in KV, so a user's open channels and regular telefunction calls stay on the same Durable Object instance.

You can also set a per-bucket scale:

```ts
const ws = telefuncWebSocket({ scale: { weur: 4, enam: 2, apac: 1 } })
```

## KV namespace

Telefunc uses a KV namespace for shard routing tokens and pub/sub presence. Add the binding to your `wrangler.jsonc`:

```jsonc
{
  "kv_namespaces": [
    { "binding": "TelefuncKV", "id": "..." }
  ]
}
```

The default binding name is `TelefuncKV`. To use a different name:

```ts
const ws = telefuncWebSocket({ kvBindingName: 'MyKV' })
```

## Custom binding name

The default binding name is `TelefuncDurableObject`. To use a different name:

```ts
const ws = telefuncWebSocket({ bindingName: 'MyDurableObject' })
```


## Location fallback

To control the canonical Durable Object placement bucket when Cloudflare cannot map the request precisely, pass `locationFallback`:

```ts
const ws = telefuncWebSocket({ locationFallback: 'weur' })
```

Telefunc first tries to resolve a canonical bucket from the incoming Cloudflare colo or continent. If that fails, it falls back to `locationFallback`, which defaults to `'weur'`.

Valid values are the canonical Telefunc buckets: `'wnam'`, `'enam'`, `'weur'`, `'eeur'`, `'apac'`, and `'oc'`.


## Hibernation

> **[Durable Object hibernation](https://developers.cloudflare.com/durable-objects/best-practices/websockets/#why-use-hibernation) while channels are open is not supported.**
>
> Channels are plain in-memory JavaScript: closures, local variables, callbacks. If the Durable Object hibernates, that heap state is gone. Telefunc therefore keeps the Durable Object alive for the lifetime of each open channel.

Once all channels are closed and no clients are connected, the Durable Object can hibernate again. By default this happens after the reconnect and idle windows expire.

Shorter reconnect and idle windows let the Durable Object hibernate sooner, but reduce tolerance for temporary disconnects.

You can shorten those windows:

```ts
const ws = telefuncWebSocket({
  reconnectTimeout: 5_000,
  idleTimeout: 5_000,
  pingInterval: 2_000,
})
```

Your own timers or background work can also keep the Durable Object awake after channels close.
