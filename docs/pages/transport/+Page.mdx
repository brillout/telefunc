**Environment**: client.

Choose the transport for <Link href="/streaming">streaming</Link> values.

```ts
// Environment: client

import { config } from 'telefunc/client'

config.transport = 'sse'
```

<ConfigWhereClient />

## Options

| Transport | Content-Type | Description |
|---|---|---|
| `'stream'` (default) | `application/octet-stream` | Raw binary chunked HTTP response. Lowest overhead. |
| `'sse'` | `text/event-stream` | Base64url-encoded Server-Sent Events. Works through proxies/CDNs that buffer binary responses but pass SSE events. |
| `'ws'` | WebSocket | Binary frames over a multiplexed WebSocket connection. See <Link href="/channels" />. |

> Plain (non-streaming) telefunction calls are always `text/plain` JSON regardless of the `transport` setting.


## Per-call override

Override the transport for a single call using <code>withContext()</code>:

```ts
import { onAIChat } from './Chat.telefunc'
import { withContext } from 'telefunc/client'

const gen = withContext(onAIChat, { transport: 'sse' })('Tell me a joke')
for await (const token of gen) {
  console.log(token)
}
```
