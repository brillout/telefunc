Telefunc handles real-time flows — chat, upload progress, live updates — using the same function model as ordinary calls.

## Which primitive should I use?

| Primitive | Best for | Setup |
|---|---|---|
| **Function passing** | Short-lived callbacks — progress updates, one-off server-to-client pushes | None |
| **Channels** | Ongoing real-time messaging — direct or broadcast | None (SSE) or <Link text={<code>tf.installWebSocket()</code>} href="/server" /> (WS) |
| **`config.stream.transport = 'channel'`** | <Link href="/stream">Streamed values</Link> over the same reconnecting transport as channels | Same as channels |


## Function passing

Telefunc can pass functions between client and server:

- **Client → server**: Pass a callback as an argument — the server can call it.
- **Server → client**: Return a function from a telefunction — call it later from the client.

### File upload progress

A callback argument reporting progress while the server reads the file stream:

```ts
// Upload.telefunc.ts
// Environment: server

import fs from 'node:fs'

export async function onUpload(file: File, onProgress: (percent: number) => void) {
  const total = file.size
  let loaded = 0

  const dest = fs.createWriteStream(`./uploads/${file.name}`)

  for await (const chunk of file.stream()) {
    dest.write(chunk)
    loaded += chunk.byteLength
    onProgress(Math.round((loaded / total) * 100))
  }

  dest.end()
}
```

```tsx
// Upload.tsx
// Environment: client

import { useState } from 'react'
import { onUpload } from './Upload.telefunc'

function Upload() {
  const [progress, setProgress] = useState(0)

  async function handleUpload(e: React.ChangeEvent<HTMLInputElement>) {
    const file = e.target.files?.[0]
    if (!file) return
    await onUpload(file, (percent) => setProgress(percent))
  }

  return (
    <div>
      <input type="file" onChange={handleUpload} />
      <div style={{ width: `${progress}%`, height: 4, background: '#3b82f6' }} />
    </div>
  )
}
```

If you only need callback-style interaction, function passing is simpler than a dedicated channel.


## Channels

Use <Link text={<code>createChannel()</code>} href="/createChannel" /> when both sides keep sending messages over time. Channels have two modes:

- **Point-to-point** (`send` / `listen`) — direct messages between one server and one client. Good for commands, request-response, or controlling a specific session.
- **Pub/sub** (`publish` / `subscribe`) — broadcast to all channels sharing a key. Good for chat rooms, live feeds, notifications.

### Chat room

```ts
// Chat.telefunc.ts
// Environment: server

import { createChannel } from 'telefunc'

type ChatMessage = { user: string; text: string; ts: number }
type Message = (msg: ChatMessage) => void

export async function onJoinChat(username: string) {
  const channel = createChannel<Message, Message>({ key: 'chat:lobby' })

  channel.onOpen(() => {
    channel.publish({ user: 'system', text: `${username} joined`, ts: Date.now() })
  })

  return { channel: channel.client }
}
```

```ts
// Chat.tsx
// Environment: client

import { onJoinChat } from './Chat.telefunc'

const { channel } = await onJoinChat('Alice')

channel.subscribe((msg) => {
  console.log(`${msg.user}: ${msg.text}`)
})

channel.publish({ user: 'Alice', text: 'Hello!', ts: Date.now() })
```

### Game session

```ts
// Game.telefunc.ts
// Environment: server

import { createChannel } from 'telefunc'

type Command = (cmd: { action: string }) => { ok: boolean }
type Event = (event: { type: string; data: unknown }) => void

export async function onJoinGame(gameId: string) {
  const channel = createChannel<Command, Event>({ ack: true })

  channel.listen((cmd) => {
    // Process player command, push game state
    channel.send({ type: 'state', data: getGameState(gameId) })
    return { ok: true }
  })

  return { channel: channel.client }
}
```

## Reconnection

If the connection drops, Telefunc reconnects automatically and resumes existing channels. Messages sent while offline are buffered and delivered in order on reconnect.

- `onOpen()` fires only for the initial open.
- `onClose()` fires only when the channel is permanently closed.

> Both sides keep a bounded replay buffer. After reconnect, missing frames are replayed and duplicates are ignored.


## Transport

By default channels use **SSE** — no extra setup required. For full-duplex WebSocket transport, call `tf.installWebSocket()` (see <Link href="/server" />).

| Transport | Throughput | Recovery | CDN-friendly | Setup |
|---|---|---|---|---|
| SSE (default) | 🟡 | ✅ | ✅ | None |
| WebSocket | ✅ | ✅ | 🟡 | <Link text={<code>tf.installWebSocket()</code>} href="/server" /> |

All channels share a single multiplexed connection per server URL.

```ts
// Environment: client

import { config } from 'telefunc/client'

config.channel.transport = 'ws'
```
