Telefunc natively supports [`File`](https://developer.mozilla.org/en-US/docs/Web/API/File) and [`Blob`](https://developer.mozilla.org/en-US/docs/Web/API/Blob) arguments. When a telefunction is called with file arguments, Telefunc automatically uses [`multipart/form-data`](https://developer.mozilla.org/en-US/docs/Web/API/FormData) instead of the usual text-based serialization. No configuration needed.

## Example

```ts
// FileUpload.telefunc.ts

export async function onFileUpload(file: File, description: string) {
  const content = await file.text()
  console.log(file.name, file.size, file.type)
  console.log(description)
  // ...
}
```

```tsx
// FileUpload.tsx

import { onFileUpload } from './FileUpload.telefunc'

async function handleSubmit(form: FormData) {
  const file = form.get('file') as File
  const description = form.get('description') as string
  await onFileUpload(file, description)
}
```

## How it works

- `File` / `Blob` arguments are sent as separate binary parts in a `multipart/form-data` request
- On the server, they are received as standard `File` / `Blob` objects
- Non-file arguments are serialized as usual (preserving `Date` objects, etc.)

## Server integration

If you use `telefunc({ url, method, body })`, pass a `FormData` as `body` for multipart requests:

```ts
const httpResponse = await telefunc({
  url,
  method: request.method,
  body: request.headers.get('content-type')?.includes('multipart/form-data')
    ? await request.formData()
    : await request.text(),
})
```

Or pass the `Request` object directly:

```ts
const httpResponse = await telefunc({ request })
```
