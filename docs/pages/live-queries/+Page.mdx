Invalidate [TanStack Query](https://tanstack.com/query) queries from the server — every connected client refetches.

## Setup

Replace `QueryClient` with `LiveQueryClient`:

```tsx
import { LiveQueryClient } from '@telefunc/tanstack-query'
import { QueryClientProvider } from '@tanstack/react-query'

const queryClient = new LiveQueryClient()

function App() {
  return (
    <QueryClientProvider client={queryClient}>
      <MyApp />
    </QueryClientProvider>
  )
}
```

`LiveQueryClient` accepts the same options as `QueryClient`.


## Example

```ts
// todos.telefunc.ts
import { invalidate } from '@telefunc/tanstack-query/server'

export async function getTodos() {
  return db.todo.findMany()
}

export async function addTodo(text: string) {
  await db.todo.create({ data: { text, done: false } })
  invalidate(['todos'])
}
```

```tsx
// Todos.tsx
import { useQuery } from '@tanstack/react-query'
import { getTodos } from './todos.telefunc'

function Todos() {
  const { data: todos } = useQuery({
    queryKey: ['todos'],
    queryFn: () => getTodos(),
  })

  return <ul>{todos?.map(todo => <li key={todo.id}>{todo.text}</li>)}</ul>
}
```

When `addTodo` runs on any client, every other client's todo list refetches.


## `invalidate(queryKey)`

```ts
import { invalidate } from '@telefunc/tanstack-query/server'

invalidate(['todos'])       // all queries starting with ['todos']
invalidate(['todos', 1])    // only ['todos', 1]
```

Prefix matching — same behavior as TanStack Query's `invalidateQueries`.


## How it works

1. `LiveQueryClient` passes each query's `queryKey` alongside the telefunc call — subscriptions only exist for queries the client successfully executed.
2. The server subscribes to invalidation events for that key.
3. `invalidate(queryKey)` publishes through <Link text="pub/sub" href="/channel#pubsub" />.
4. The client receives the event and calls `invalidateQueries()`.

Works across multiple servers via the <Link text="pub/sub adapter" href="/channel#multi-server" />.
