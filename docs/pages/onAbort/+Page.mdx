**Environment**: client.

The `onAbort()` hook is called whenever the browser makes a telefunction call that fails because the telefunction ran <Link text={<code>throw Abort()</code>} href="/Abort" />.

It does **not** run when other errors are thrown, see <Link href="/abort-vs-error" />.

```ts
// Environment: client

import { onAbort } from 'telefunc/client'

onAbort((err) => {
  if (err.abortValue.notLoggedIn) {
    // Redirect user to login page
    window.location.href = '/login'
  }
})
```

> The `onAbort()` hook is usually used for implementing `getContext()` permission wrappers, see <Link href="/permissions#getcontext-wrapping" doNotInferSectionTitle={true} />.


## `abortValue`

When the server uses `throw Abort(someValue)`, the value is available on `err.abortValue`:

```ts
import { Abort } from 'telefunc/client'

onAbort((err) => {
  // err instanceof Abort === true
  console.log(err.abortValue) // The value passed to Abort()
})
```

This works for both regular and streaming telefunctions. For streaming, the hook fires when a generator calls `throw Abort(value)` mid-stream — see <Link href="/Abort#mid-stream-abort" doNotInferSectionTitle={true} />.
