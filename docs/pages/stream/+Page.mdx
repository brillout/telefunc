Telefunctions can stream values to the client using `AsyncGenerator`, `ReadableStream`, or nested `Promise`s.

> Streaming works out of the box with zero config. Plain telefunction calls are unaffected.


## Which one should I use?

| Type | Best for | Values |
|---|---|---|
| [`AsyncGenerator`](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/AsyncGenerator) | AI tokens, notifications, progress updates | Structured (JSON-serializable) |
| [`ReadableStream`](https://developer.mozilla.org/en-US/docs/Web/API/ReadableStream) | File downloads, media, raw bytes | Binary (`Uint8Array`) |
| Nested [`Promise`](https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Promise) | Lazy parts of a response that resolve independently | Any serializable value |


## `AsyncGenerator`

The usual choice for incremental structured values.

```ts
// Chat.telefunc.ts
// Environment: server

export async function* onChat(prompt: string): AsyncGenerator<string> {
  const stream = await ai.streamText({ prompt })
  for await (const token of stream) {
    yield token
  }
}
```

```tsx
// Chat.tsx
// Environment: client

import { onChat } from './Chat.telefunc'

const gen = onChat('Tell me a joke')
for await (const token of gen) {
  console.log(token) // Each token arrives as it's yielded
}
```


## `ReadableStream`

Use `ReadableStream` when you need to stream bytes.

```ts
// Download.telefunc.ts
// Environment: server

import fs from 'node:fs'
import { Readable } from 'node:stream'

export function onDownload(fileName: string): ReadableStream<Uint8Array> {
  return Readable.toWeb(fs.createReadStream(`./files/${fileName}`)) as ReadableStream<Uint8Array>
}
```

```ts
// Download.tsx
// Environment: client

import { onDownload } from './Download.telefunc'

const stream = await onDownload('report.pdf')
const reader = stream.getReader()
while (true) {
  const { done, value } = await reader.read()
  if (done) break
  // Process raw bytes
}
```

## Nested `Promise`

Return unawaited promises inside the response object — the client gets the surrounding values immediately while each promise resolves independently.

```ts
// Dashboard.telefunc.ts
// Environment: server

export async function onLoadDashboard() {
  return {
    title: 'Dashboard',            // Delivered immediately
    summary: fetchSummary(),        // Resolves independently
    report: fetchExtensiveReport(), // Resolves independently
  }
}
```

```tsx
// Dashboard.tsx
// Environment: client

import { onLoadDashboard } from './Dashboard.telefunc'

const res = await onLoadDashboard()

console.log(res.title)             // Available right away

const summary = await res.summary  // Await when needed
const report = await res.report
```

> **Don't `await` inside the telefunction** — that blocks the entire response:
> ```ts
> // ❌ Blocks everything
> export async function onLoadDashboard() {
>   await prepare()
>   return { title: 'Dashboard' }
> }
> ```
> Return the promise instead:
> ```ts
> // ✅ title delivered immediately, promise resolves in the background
> export async function onLoadDashboard() {
>   return {
>     title: 'Dashboard',
>     data: prepare(),
>   }
> }
> ```


## Mixed return types

A telefunction can return any combination of generators, streams, and promises.

```ts
// Dashboard.telefunc.ts
// Environment: server

export async function onLoadDashboard() {
  async function* notifications(): AsyncGenerator<string> {
    for (const n of await fetchNotifications()) {
      yield n.text
    }
  }

  return {
    title: 'Dashboard',
    stream: notifications(),
    fast: Promise.resolve({ count: 42 }),
    slow: fetchExtensiveReport(),
  }
}
```

```tsx
// Dashboard.tsx
// Environment: client

import { onLoadDashboard } from './Dashboard.telefunc'

const res = await onLoadDashboard()

// Static values — available immediately
console.log(res.title)

// Consume generator and promises independently
for await (const notification of res.stream) {
  console.log(notification)
}

const fast = await res.fast
const slow = await res.slow
```

> Multiple generators and promises resolve concurrently without blocking each other.


## Cancellation

Stop consuming a stream gracefully from the client using `close()`, `break`, `gen.return()`, or `reader.cancel()`. All of these cleanly signal the server to stop producing and release resources.

### `close()`

`close()` from `telefunc/client` gracefully closes any streaming value — generators, streams, channels, and passed functions. It walks the return value recursively and closes everything in one call.

```ts
// Environment: client

import { close } from 'telefunc/client'
import { onLoadDashboard } from './Dashboard.telefunc'

const result = await onLoadDashboard()

// Read what you need...
const first = await result.stream.next()

// Then close everything at once
close(result)
```

### `break` / `gen.return()`

Breaking out of a `for await` loop implicitly calls `gen.return()`, which gracefully stops the generator:

```ts
const gen = onChat('Tell me a story')
for await (const token of gen) {
  if (shouldStop) break // Gracefully stops the server-side generator
}
```

You can also call `gen.return()` directly:

```ts
const gen = onChat('Tell me a story')
const first = await gen.next()
gen.return() // Gracefully stops
```

### `reader.cancel()`

For `ReadableStream`, call `cancel()` on the reader:

```ts
const stream = await onDownload('large-file.zip')
const reader = stream.getReader()
const first = await reader.read()
reader.cancel() // Gracefully stops the server-side stream
```


## Cleanup

Use `onClose()` on the server to detect when the response ends — whether the stream finished, the client disconnected, or an error occurred:

```ts
// Environment: server

import { onClose } from 'telefunc'

export async function* onChat(prompt: string) {
  onClose(() => {
    // Release resources, cancel upstream work
  })

  for await (const token of ai.streamText({ prompt })) {
    yield token
  }
}
```

## Transport

By default, streamed values are sent as raw binary chunks over HTTP (`application/octet-stream`). You can switch transport globally or per call.

| Transport | Description |
|---|---|
| `'binary-inline'` (default) | Raw binary chunked HTTP response. Lowest overhead. |
| `'sse-inline'` | Base64url-encoded Server-Sent Events — works through proxies that buffer binary |
| `'channel'` | Delivers over the configured channel transport (`'sse'` or `'ws'`) with reconnection support |

```ts
// Environment: client

import { config } from 'telefunc/client'

config.stream.transport = 'sse-inline'
```

Override per call:

```ts
import { onChat } from './Chat.telefunc'
import { withContext } from 'telefunc/client'

const gen = withContext(onChat, {
  stream: { transport: 'channel' },
})('Tell me a joke')
```

See <Link href="/transport" /> for a full comparison.


## Error handling

Telefunc distinguishes expected failures from bugs — even in streaming responses. If a generator or stream throws, the client receives a typed error while values already delivered remain available.
