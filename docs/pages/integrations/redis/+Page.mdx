Redis-backed scaling for Telefunc — pub/sub fan-out across instances, plus cross-instance routing for channels.

## Install

```bash
npm install @telefunc/redis ioredis
```

## Setup

```ts
import IORedis from 'ioredis'
import { installRedis } from '@telefunc/redis'

const redis = new IORedis('redis://localhost:6379')
installRedis(redis)
```

That configures Redis for Telefunc's cross-instance realtime features:

- publishes from any instance fan out to every subscriber instance
- channels route correctly when their connection lands on a different instance behind your load balancer

### Sharing an existing client

Pass an existing [`ioredis`](https://github.com/redis/ioredis) Redis or Cluster instance when you want to share a connection or set custom options such as TLS or retry strategy.

```ts
import IORedis from 'ioredis'
import { installRedis } from '@telefunc/redis'

const redis = new IORedis(process.env.REDIS_URL, { tls: {} })
installRedis(redis)
```

### What it configures

`installRedis()` wires two pieces at once:

- pub/sub fan-out across instances
- cross-instance channel routing

That means Redis covers both broadcast messaging and channels whose runtime state lives on a different server instance.

All subscribers observe the same publish order for a given key.
